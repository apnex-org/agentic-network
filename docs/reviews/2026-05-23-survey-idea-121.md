## §5 Design-phase Inputs (load-bearing for Phase 4)

### 5.1 Architect-side Design surface

- **Verb-vocabulary classification doc** — enumerate ALL current MCP tools + classify per verb-namespace; identify renames needed (`get_task` → `claim_task`; `drain_pending_actions` → `pull_next_action`; etc.)
- **Deprecated-tool inventory** — list all tools to remove pre-Phase-A (output of bilateral Design discovery)
- **Per-kind intent/workflow verbs** — for each of the 20 kinds (post idea-126 cutover), enumerate allowed actions (e.g., Thread: `claim_`/`close_`; Mission: `propose_`/`acknowledge_`; Bug: `resolve_`)
- **CLI scripts verb-discipline audit** — `scripts/local/*.sh` survey; rename where misaligned

### 5.2 Engineer-side Design surface

- **Pre-commit linter** — `scripts/lint/verb-namespace.sh` or similar; static check on tool registrations
- **Runtime registration validation** — Hub startup-time gate on tool namespace conformance
- **`get_resource` polymorphic dispatcher** — id-prefix routing to kind-specific repository read
- **Tool deprecation mechanics** — how deprecated tools surface to LLM (descriptions? dynamic scope per idea-115? hard exit?)
- **CLI script migration** — bash + node script verb-namespace conformance

### 5.3 Composition checkpoints

- **idea-126 SchemaDef v2.0** — single source of truth; idea-121 consumes the spec/status JSON Schemas to generate tools per kind
- **idea-126 `spec.verbs` per-kind declaration** — Q2-D deferral lands here; intent/workflow verbs catalogued in SchemaDef v2.0; idea-121 tool-surface generated from declaration
- **`get_resource({view: "flat"|"full"})` projection layer** — composes idea-126's strict-K8s envelope with idea-121's LLM-ergonomic flat shape (per thread-634 Q8 architect counter-proposal accepted)

### 5.4 Acceptance criteria

- Pre-commit linter blocks new tool registrations outside verb-namespace
- Runtime registration validation rejects non-conforming tools at Hub startup
- `get_resource({id})` polymorphic id-prefix dispatch covers all 20 kinds post-idea-126
- `get_resource({view: "flat"})` returns LLM-ergonomic flat shape; `view: "full"` returns envelope
- All deprecated tools removed pre-Phase-A; clean slate verified via post-cleanup tool-catalog enumeration
- Legacy `get_<entity>` (13 tools) removed at Phase B big-bang
- Intent/workflow verbs declared in SchemaDef v2.0 + corresponding tools live at Phase B
- All CLI scripts (`scripts/local/*.sh`) verb-namespace-conformant post-Phase-A
- Tool-catalog context-burn metrics (tele-10) measurably reduced
