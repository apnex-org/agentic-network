## §5 Design-phase Inputs (load-bearing for Phase 4)

### 5.1 Architect-side Design surface

- **Relationship SchemaDef** — added to `hub/scripts/entity-kinds.json` post-idea-126 cutover; inherits envelope shape
- **Cascade-action ↔ edge-type mapping enumeration** (Q3-D deferral) — for each existing cascade-action handler, declare which edge-type to emit:
  - `create_idea` cascade → `Idea derives_from Thread`
  - `propose_mission` cascade → `Mission derives_from Idea`
  - `create_bug` cascade → `Bug derives_from Thread`
  - `update_idea(status=incorporated, missionId)` → `Idea references Mission`
  - (additional mappings enumerated bilateral at Phase 4)
- **Cardinality rules** — per-edge-type semantics (some 1-1 like `supersedes`, others M-N like `depends_on`); declare in SchemaDef or enforce at cascade-handler

### 5.2 Engineer-side Design surface

- **Postgres table schema** — `relationships` with envelope JSONB; primary key `id`; indexed on `(fromId, edgeType)` for forward traversal
- **Substrate-level indexed reverse view** — `reverse_relationships` JSON-indexed projection; maintained at write-time by substrate (not application layer)
- **`create_relationship` MCP tool** — manual edge-creation per idea-121 verb-discipline (lifecycle verb)
- **Cascade-handler integration** — handlers gain edge-emit machinery per the Design-enumerated mapping
- **Graph-walk surface** — `list_relationships({filter: {fromId, edgeType}})` + `list_relationships({filter: {toId, edgeType}})` queries; recursive walk deferred to idea-151 v2 if patterns warrant

### 5.3 Composition checkpoints

- **idea-126 SchemaDef v2.0** — Relationship inherits the substrate-wide envelope; kind 21 added post-cutover
- **idea-121 verb-tool surface** — `create_relationship` / `list_relationships` / `get_resource({id: "relationship-..."})` consume the SchemaDef
- **W4 cascade-FSM** — Relationship edges enable:
  - idea-93 deferred-cascade queue (`depends_on` for action ordering)
  - idea-96 retry chains (`derives_from` lineage)
  - Supersession patterns (`supersedes` enables stale-action skipping)
  - bug-23 bilateral-seal race (cascade-spawn `references` + `derives_from` ordering)

### 5.4 Acceptance criteria

- `Relationship` SchemaDef kind added (kind 21); envelope shape per idea-126 substrate
- `relationships` table created in postgres; forward-only edge storage
- `reverse_relationships` substrate-level indexed view operational
- `create_relationship` MCP tool live; manual edge-creation works
- Cascade-action ↔ edge-type mapping enumerated + implemented per Design enumeration
- All v1 cascade-handlers auto-emit appropriate edges per mapping
- Graph-walk queries (`list_relationships` forward + reverse) deterministic and indexed-cheap
- W4 cascade-FSM downstream missions (idea-93/96, bug-23) absorb Relationship machinery without code-level edge-creation
