## 1. Roller architecture  [RUBY]

The roller is host-side bash (like startup.sh's `refresh.sh` and the proven ¬ßB.3 manual roll), NOT a containerized agent ‚Äî no bootstrap image, and it mirrors the existing codified `refresh-docker-token.*` pattern. Files install under `/var/lib/hub-roller/` (COS `/var` is writable but `noexec` ‚Üí invoke via `/bin/bash`, as `refresh-docker-token.service` does), each **pinned by content SHA-256** asserted at install and **re-asserted before every use (R8)** ‚Äî the roller AND boot (¬ß9.1) hash-check each script BEFORE sourcing/exec, fail-closed on drift:
- `roller.sh` ‚Äî tick body (resolve ‚Üí detect ‚Üí validate ‚Üí roll ‚Üí verify ‚Üí receipt);
- `hub-resolve.sh` ‚Äî the SHARED gcloud-free resolver (roller **and** boot);
- `hub-launch.sh` ‚Äî the SHARED Hub-ONLY launcher (roller **and** boot).
Auth: `DOCKER_CONFIG=/var/lib/hub/docker-config` (the **literal** path ‚Äî startup.sh:45-47 sets it from `$HUB_DIR`, but a systemd unit does NOT shell-expand `$HUB_DIR`, so the unit hard-codes the literal ‚Äî S4) ‚Üí `docker-credential-gcr` ‚Üí metadata ‚Üí `hub-vm-sa@` (`artifactregistry.reader`) ‚Äî the PROVEN ¬ßB.3 path. Contrast the retired watchtower: its OWN registry client couldn't use the CLI helper ‚Üí the parser bug + bug-107 static-token race; the roller uses the CLI and sidesteps both.
**Runtime compatibility baseline (S4/T5 ‚Äî a REVIEWED RECEIPT, not trust-on-first-use):** the Hub VM's Terraform uses the **floating** `cos-cloud/cos-stable` family; this design does NOT claim Terraform pins an exact image (changing the boot image would force a VM REPLACEMENT, which the cutover forbids). Instead, BEFORE the implementation gate a **read-only production observation receipt** is captured + **reviewer-accepted**, binding the EXACT: `/etc/os-release` fields (`ID`/`VERSION`/`BUILD_ID`), the corresponding **GCE boot-image locator** (`projects/cos-cloud/global/images/<exact>`), `docker` + `systemd` versions, the `docker-credential-gcr` helper **path + version + SHA-256**, the **python3 path + version + SHA-256** (the ¬ß1.8 durability helper), and the presence of `bash`/`flock`/`curl`/`tar`/`jq`. The roller's installer constants are BOUND to this receipt; at start it asserts the LIVE runtime == the receipt values and **fails DISABLED** on any mismatch / absence / drift ‚Äî never dynamic capture-and-trust. The exact COS-GCE sandbox (greg ¬ß5) runs the SAME reviewer-accepted identities. **Future VM-recreate pinning** (pinning `cos-stable` to that exact image locator) is a SEPARATE change, out of the zero-replacement cutover scope.

Each cell below is a **TOTAL wall bound** (kill-grace already counted). The service unit provides the dir via `RuntimeDirectory=hub-roller` + `RuntimeDirectoryMode=0700`. **Flock scope (R5):** excludes only concurrent *roller* instances ‚Äî NOT a manual/root docker command; the no-manual-writer claim is evidence-bounded (¬ß1.7 + greg ¬ß4), not proven by the lock.

### 1.3 SHARED resolver `hub-resolve.sh` ‚Äî TYPED outcome (R1) + FROZEN method (R4)
Wraps greg's `hub-detect` (seam: `hub-detect` emits a raw `$REG@sha256:<64hex>` on stdout, exit 0 on a clean single resolution, nonzero on any ambiguity/failure/no-auth). The resolver returns a **TYPED outcome** ‚Äî `fresh_candidate` | `boot_fallback` | `quarantined` | `error` ‚Äî plus the exact digest for the first two. It NEVER emits a mutable tag; the CALLER's policy decides what each outcome permits.

**FROZEN detect method (R4)** ‚Äî no (i)/(ii)/(iii) open selection: `docker pull "$REG:latest"` (CLI + gcr-helper) ‚Üí `docker image inspect` the pulled image ‚Üí require **exactly one** canonical `$REG@sha256:[0-9a-f]{64}` member of `.RepoDigests` (zero/multiple ‚áí `error`, never index-`[0]`) ‚Üí that captured digest is the candidate. (Direct registry-`HEAD`-via-helper-token and `docker manifest inspect` are OUT ‚Äî unproven / conflict with the no-assumed-helper-semantics rule.) The sandbox (greg ¬ß5) proves this exact CLI path.

**Outcome logic:**
1. `hub-detect` clean + digest validates + not quarantined ‚áí `fresh_candidate(digest)`.
2. resolved digest == persisted `D_bad` and still-current ‚áí `quarantined`.
3. `hub-detect` fails/ambiguous/no-auth ‚áí `boot_fallback(lastGoodDigest)` if a valid persisted `lastGoodDigest` exists, else `error`.

**CALLER POLICY (R1 no-backwards-roll + S1 boot FSM):**
- **Periodic roller tick** advances (rolls) **ONLY** on `fresh_candidate`. `boot_fallback` / `quarantined` / `error` ‚áí **NO-OP** ‚Äî a tick must NEVER roll to `lastGoodDigest`; a transient registry failure must not let a stale fallback read as "new-vs-running" and roll production backwards.
- **Boot FSM (S1 ‚Äî never a raw resolver digest straight to the launcher):**
  1. valid persisted `lastGoodDigest` exists ‚áí launch THAT exact known-good digest (already provenance+health-verified when it was rolled ‚Äî no re-validation needed); the periodic tick later picks up any newer `fresh_candidate` through the full validate‚Üíverify‚Üírollback path.
  2. else (cold start / recreated VM, no last-good) ‚áí a `fresh_candidate` may be launched ONLY after passing the SAME ¬ß3 no-exec provenance validation AND a post-launch bounded health check; if either fails ‚áí **loud manual state** (there is NO rollback target at cold-start). A merely regex-valid fresh digest is NEVER passed resolver‚Üílauncher.
  3. `error` / `quarantined` at boot with no valid `lastGoodDigest` ‚áí fail-loud to manual (no launch target ‚Äî never `:latest`).
- **Trust boundary (S1/¬ß3, stated honestly):** a candidate's AUTHORIZATION comes from the **exact AR repo + its writer IAM**, which authorizes **ORG-APPROVED WRITERS** ‚Äî CI **and** the governed manual BUILD ‚Äî to push there (NOT "only CI"). build-info format + `/health` gitSha prove **compatibility + identity** (a well-formed image, which gitSha it is) ‚Äî NOT that an arbitrary gitSha is an approved commit. Provenance/health is a compatibility+identity gate, not an authorization oracle; for `ci_greenrun`, the exact CI-run + `D_src` equality binding is what proves a CI origin specifically (greg ¬ß3 / verify_autoroll), and stays INDEPENDENTLY required.

### 1.4 SHARED launcher `hub-launch.sh $REG@sha256:...` ‚Äî Hub-ONLY, exact-digest-only (steve)
- **Arg contract (steve):** accepts ONLY an exact `$REG@sha256:[0-9a-f]{64}` ref; **fails closed on any tag** (regex-assert the arg; a `:tag`/bare/mutable ref aborts nonzero before touching docker).
- Recreates ONLY `ois-hub-prod` from the `startup.sh:107-119` **shared snippet** (one source, no line-range drift). **PREFLIGHT before ANY mutation (T6/U2) ‚Äî under the INHERITED single-owner lock** (the caller ‚Äî the roller tick (¬ß1.2) or the ¬ß9.1 boot controller ‚Äî acquired `/run/hub-roller/lock` EXACTLY ONCE on the fixed fd; `hub-launch.sh` ASSERTS the inherited fd (`/proc/self/fd` ‚Üí `/run/hub-roller/lock`), it NEVER closes/reopens/re-acquires the flock), run ALL non-mutating prerequisites: validate the exact-ref arg (`$REG@sha256:‚Ä¶`); verify the gcr-helper + `DOCKER_CONFIG` readable; mint a fresh metadata token; fetch + validate every Secret-Manager secret; and **ensure the exact target digest is LOCALLY AVAILABLE via an authenticated exact-digest `docker pull "$1"`** (critical for boot-fallback, whose last-good image may have been GC'd). These prove exact-ref/auth/secrets/**pullability** ‚Äî **NOT** health, which is necessarily POST-launch (¬ß1.5 verify/rollback), never a pre-mutation proof (U4). ONLY after every preflight step passes: `docker rm -f ois-hub-prod`; `docker run -d --name ois-hub-prod --restart unless-stopped --network hub-net -p 8080:8080 -e NODE_ENV=production -e PORT=8080 -e POSTGRES_CONNECTION_STRING=‚Ä¶ -e HUB_API_TOKEN=‚Ä¶ -e WATCHDOG_ENABLED=true -e HUB_ADMIN_TOKEN=‚Ä¶ -e OIS_GH_API_TOKEN=‚Ä¶ -e OIS_REPO_EVENT_BRIDGE_REPOS=‚Ä¶ "$1"`. A preflight failure aborts nonzero **with the current Hub still running** (never a remove-then-fail gap). **The `com.centurylinklabs.watchtower.enable` label is REMOVED (R7)** ‚Äî retirement is fail-safe by construction: any watchtower that ever reappears finds no opted-in container.
- Touches NOTHING else: no `ois-postgres-prod`, no `hub-backup`, no watchtower, no full startup runner.
- Consumed by BOTH: boot (`startup.sh` ‚Üí `hub-resolve.sh` ‚Üí `hub-launch.sh "$D_boot"`) and the roller (`hub-launch.sh "$D_new"` / `"$D_prev"`). startup.sh stops inlining the docker-run and **never** launches by `:latest`.
- **Fresh token + secrets on EVERY invocation (S5):** `hub-launch.sh` runs `set -euo pipefail`; on EVERY launch/rollback it INDEPENDENTLY mints a **fresh metadata access token** (the boot-minted token is stale hours later ‚Äî a roller launch cannot reuse it) and **re-fetches + validates ALL required secrets** from Secret Manager ‚Äî never a cached/persisted value, so a rotated secret is picked up and no stale one is launched. Fail-closed: any non-200 HTTP, malformed/empty JSON, empty/invalid secret, or a `DOCKER_CONFIG` unreadable by the service identity ‚áí abort nonzero BEFORE `docker run` (no partial launch). No token/secret is ever logged (greg ¬ß8 sentinel-tested).

### 1.5 The roll ‚Äî embeds ROLL v8 engine (@515bd02), generalized to D_new/D_prev
Per tick, after resolve+detect find a NEW valid candidate `D_new` ‚âÝ running (provenance-validated per greg ¬ß3):
1. `D_prev` = running `ois-hub-prod` RepoDigest **before any mutation**, **validated as exactly one canonical `$REG@sha256:[0-9a-f]{64}` membership before it is accepted as the rollback target (S7)** ‚Äî the DYNAMIC rollback target (not a fixed digest; corrects #629). If `D_prev` is absent/ambiguous, the roll does NOT proceed (no safe rollback target).
2. **PG pre-snapshot:** `ois-postgres-prod` container ID + mount (attached PD) + liveness.
3. `hub-launch.sh "$D_new"` (v8 `recreate_hub`; `$D_new` / `$D_prev` / `lastGoodDigest` are already FULL canonical `$REG@sha256:‚Ä¶` refs ‚Äî passed to the launcher directly, NEVER re-prefixed with `$REG@`).
4. **VERIFY** (v8 `verify` = `health_local`/`health_ext`/`sha_of` + RepoDigest membership): **FROZEN bounded health-poll ‚Äî every 5s up to a 120s window (‚â§24 attempts)** (greg ¬ß4/¬ß6 reference this window; ¬ß1.2 ‚Äî the single timeout owner ‚Äî derives its counted `TimeoutStartSec=2200`/`TimeoutStopSec=1500` from this window) ‚Äî local + external cache-busted `/health` gitSha == D_new build-info gitSha **AND** running RepoDigest membership == `$D_new`. Window-exceeded ‚áí treat as verify-FAIL ‚Üí rollback.
5. **verify OK ‚áí** persist `lastGoodDigest = D_new` (atomic, ¬ß1.8); emit `rolled` receipt (disposition `roll_success`).
6. **verify FAIL / timeout ‚áí ROLLBACK:** `hub-launch.sh "$D_prev"`; verify running == D_prev (same 120s/5s window); **persist `D_bad = D_new`** (atomic quarantine, ¬ß1.8 ‚Äî ¬ß2/¬ß6 circuit-breaker); emit `rolled_back` (v8 rc2) ‚Äî or `rollback_unproven` (v8 rc3, fail-loud) if the rollback itself won't verify. **rc2 is NEVER forward-success.** **Quarantine ALSO ingests a candidate REJECTED by ¬ß3 provenance (S7)** ‚Äî not only health-fail rollbacks ‚Äî so the same invalid `:latest` is not re-pulled/re-validated every cadence (greg ¬ß3 seam).
7. **PG invariant (post):** assert `ois-postgres-prod` ID + mount + liveness UNCHANGED vs the pre-snapshot; any mismatch ‚áí fail-loud (the roll must never touch PG or its PD).
Receipts: create-once JSON under `/var/lib/hub-roller/receipts/<ts>.json` (BUILD-capture discipline; atomic write per ¬ß1.8) binding greg's ¬ß4 causality fields + PG invariants + helper SHA + the `lastGoodDigest`/`D_bad` transitions; **no secrets** (greg ¬ß8 sentinel-tested).

### 1.6 Install / disable / rollback ‚Äî REBOOT-DURABLE safe-manual, ORDERED (R2 + S3)
- **INSTALL:** write the 3 scripts + 2 units (exact bytes, SHA-256 asserted); `systemctl daemon-reload`. **Enable is gated by a Terraform-controlled metadata flag `hub-roller-enabled` (¬ß9):** startup.sh always installs the units but only `systemctl enable --now hub-roller.timer` when the flag reads exactly `true`.
- **ROLLBACK = REBOOT-DURABLE SAFE-MANUAL, ORDERED (R2 + S3):** the naive "disable timer + remove scripts" is NOT reboot-durable ‚Äî unchanged metadata would reinstall/re-enable the failed roller on the next boot. The rollback is an ORDERED, verified transition:
  1. **live-disable FIRST:** `systemctl disable --now hub-roller.timer` (roller stops rolling immediately).
  2. **apply the exact false plan:** `hub-roller-enabled` `true‚Üífalse` via the ¬ß9.2 predicate.
  3. **verify metadata == `false`** (read back the applied metadata).
  - **Failure posture:** if step 2 or 3 fails or cannot be verified ‚áí the rollback is **INCOMPLETE** ‚Äî declare it so, **PROHIBIT reboot** (a reboot could re-enable the roller), keep the timer disabled, and **ESCALATE**. NEVER claim safe-manual / reboot-durable until BOTH the live timer AND the authoritative metadata flag are verified disabled. It NEVER restores the old watchtower metadata.

### 1.8 State-integrity contract (R6) ‚Äî `lastGoodDigest`, `D_bad`, receipts
All persisted roller state lives on the **persistent boot disk** under `/var/lib/hub-roller/` (NOT tmpfs `/run`, which holds only the lock) so it survives container restarts; each item obeys:
- **Durability primitive ‚Äî write-all + TWO modes (T7/U3; ONE frozen python3 helper; python3 path+version+SHA bound in the ¬ß1.1/T5 receipt, fail-DISABLED if absent/drifted):** COMMON temp write ‚Äî `fd = os.open(tmp, os.O_CREAT|os.O_EXCL|os.O_WRONLY|os.O_NOFOLLOW, 0o600)` in the SAME `0700` dir (the `O_CREAT|O_EXCL|O_NOFOLLOW` create ‚Äî never a shell redirection ‚Äî IS the implementable no-follow + no-clobber) ‚Üí **write-all loop** (`os.write` may short-write; loop until every byte is written; on ANY error, `os.close`+`os.unlink(tmp)` and fail) ‚Üí `os.fsync(fd)` ‚Üí `os.close(fd)`. Then by mode:
  - **REPLACE-STATE** (`lastGoodDigest`, `D_bad` ‚Äî replaceable): `os.replace(tmp, target)` (same-dir atomic rename, overwrite OK) ‚Üí `dfd = os.open(parent, os.O_DIRECTORY)` ‚Üí `os.fsync(dfd)` ‚Üí `os.close(dfd)`.
  - **CREATE-ONCE RECEIPT** (never clobber): `os.link(tmp, target, follow_symlinks=False)` ‚Äî **reject EEXIST** (target already exists) ‚Üí `os.unlink(tmp)` ‚Üí `os.fsync` the parent dir. A lost/partial receipt NEVER affects launch safety but **BLOCKS evidence completion**.
  A reader always sees a whole prior or whole new value; the name update is crash-durable. Mutation-tested (greg ¬ß6): temp-symlink (O_EXCL/O_NOFOLLOW rejects), short-write, torn write, missing parent-dir fsync, receipt-target clobber (EEXIST rejected).
- **Perms/ownership:** `0600`, owned by the service user; parent dir `0700`.
- **Schema/version:** a `{schema_version, ‚Ä¶}` envelope; an unknown/newer version is treated as unreadable (fail-closed), never guessed.
- **Validation on READ:** `lastGoodDigest`/`D_bad` must match `^<REG>@sha256:[0-9a-f]{64}$` (exact registry prefix + digest regex); anything else ‚áí treated as ABSENT/corrupt. **A corrupt/invalid/tampered state value must NEVER become a launch or rollback target** ‚Äî boot with no valid `lastGoodDigest` fails-loud to manual; a corrupt `D_bad` degrades safe (quarantine nothing, but a roll still requires `fresh_candidate`).
- **Symlink safety (S7/T7):** implemented BY the primitive's `O_CREAT|O_EXCL|O_NOFOLLOW` create under the `0700` root-owned dir (above) ‚Äî NOT a raceable shell `[ -L ]` test.
- **VM recreation:** a freshly-recreated VM has an empty `/var/lib/hub-roller` ‚Üí boot resolves a `fresh_candidate` or fails-loud to manual (explicit; never launches an unvalidated ref). Receipts are best-effort durable; their loss never affects launch safety.

### 9.2 Terraform apply ‚Äî reboot durability (steve pt-1 discipline)
startup.sh + the `hub-roller-enabled` flag both live in VM metadata via `modules/hub/compute.tf` (`metadata_startup_script = file(".../startup.sh")`; `metadata = { hub-roller-enabled = ... }`). Every plan/apply below: `terraform plan -out=<planfile>` (kept LOCAL/protected) ‚Üí SHA-256 the planfile ‚Üí `terraform show -json <planfile>` ‚Üí assert **ZERO resource replacement** (no `-/+`, metadata is mutable/non-replacement-forcing) ‚Üí `terraform apply <planfile>` (the EXACT saved plan) under **state-lock** ‚Üí record **pre/post metadata hashes** + re-check **state lineage + serial**.
**Three EXACT plan predicates (S2 ‚Äî the allowlist depends on the transition):**
1. **initial cutover:** the ONLY changes are the approved `metadata_startup_script` bytes **and** `hub-roller-enabled` set to exact `false` (both in-place metadata; nothing else).
2. **enable (¬ß9.4 S4):** the ONLY change is `hub-roller-enabled` `false‚Üítrue`.
3. **rollback (¬ß1.6):** the ONLY change is `hub-roller-enabled` `true‚Üífalse`.
Any plan showing more than its predicate's allowed change ‚áí **REJECT** (do not apply). Metadata reads of the flag parse an **exact `true|false`**; missing/invalid ‚áí default **disabled**.
- **Plan secret-handling (steve):** the saved plan is potentially SECRET-BEARING ‚Üí keep it LOCAL/protected; **publish only** its SHA-256 + a **redacted machine-readable change summary** (`terraform show -json` filtered to resource addresses + change actions, **no values**). **Bind provenance:** exact Terraform commit, `.terraform.lock.hcl` provider versions, workspace + backend, var-file hash, and state **lineage + serial**. The cutover receipt carries NONE of the plan and NO secret ‚Äî only these hashes/refs.

### 9.3 Live imperative removal ‚Äî apply ‚âÝ live-stop (steve pt-2)
The in-place metadata change affects only FUTURE boots; it does NOT stop the RUNNING watchtower/timer. So on the live VM the cutover ALSO:
- **Capture prior state** (reversible-cutover receipt): `docker inspect watchtower-prod`; `systemctl status refresh-docker-token.{service,timer}`.
- `docker stop watchtower-prod && docker rm watchtower-prod`.
- `systemctl disable --now refresh-docker-token.timer`; remove the `refresh-docker-token.service`/`.timer` units + `refresh.sh`; `systemctl daemon-reload`.
- **VERIFY absent:** `docker inspect watchtower-prod` fails; `systemctl status refresh-docker-token.timer` gone.

### 9.4 Cutover FSM ‚Äî NO two-writer overlap, per-step compensation (R3 + R8)
The **one-writer invariant** dominates zero-gap availability: watchtower is removed BEFORE the roller is enabled ‚Äî never both enabled at once (`flock` does not coordinate with legacy watchtower). A bounded, auditable manual-only gap between S2 and S4 is acceptable; overlapping writers are not. Each state has a defined compensation; any failure lands in a **verified manual-only** state, never an ambiguous half-install.

| State | Action | Compensation on failure |
|---|---|---|
| **S0** | Install the roller on the live VM **DISABLED** (write + SHA-assert scripts/units, `daemon-reload`; timer NOT enabled) **AND SEED initial `lastGoodDigest` (T3):** validate the CURRENTLY-running `ois-hub-prod` digest as exactly one canonical `$REG@sha256` membership with coherent build-info/health ‚Üí atomically persist it (¬ß1.8) as the initial `lastGoodDigest`, **BEFORE any metadata change** | install/assert/seed fails ‚áí remove partial install; NO metadata applied; watchtower still sole writer (safe) |
| **S1** | Terraform `plan -out`‚Üí`apply` the future-boot metadata (startup.sh bytes + `hub-roller-enabled=false`). Because S0 already seeded `lastGoodDigest`, a reboot in ANY subsequent window launches that pre-cutover known-good via the ¬ß9.1 boot controller ‚Äî **never** the cold `:latest` path | apply fails/rejected ‚áí abort; the seeded state is harmless; watchtower still sole writer |
| **S2** | Capture WT prior-state (¬ß9.3) ‚Üí `docker stop && rm watchtower-prod`; `systemctl disable --now refresh-docker-token.timer` + remove its units/`refresh.sh`; `daemon-reload`; **VERIFY absent** | **compensate by OBSERVED state (S6):** if NO retirement mutation has landed yet ‚áí converge to verified WT-only; if ANY mutation landed (container stopped/removed OR timer disabled) ‚áí drive to verified MANUAL-only (finish the removal, VERIFY absent) ‚Äî never a half-removed WT; roller stays DISABLED throughout |
| **S3** | Run ONE explicitly **non-mutating detect-only canary** (greg ¬ß7): resolve+detect+validate+LOG, **no roll**; confirm gcr-helper auth + frozen detect + provenance in prod | canary fails ‚áí roller stays DISABLED; Hub already on a good digest ‚Üí **bounded manual-only** (auditable); fix + retry S3, never enable on a failed canary |
| **S4** | Enable active rolling: flip `hub-roller-enabled=true` via the exact `plan -out`‚Üí`apply` transition **and** `systemctl enable --now hub-roller.timer` | enable fails ‚áí set flag false + disable timer ‚áí manual-only |

- **Invariant:** at NO state are two writers both enabled ‚Äî WT is gone (S2) strictly before the roller is enabled (S4).
- **Idempotent + fail-loud + OBSERVED-state compensation (S6):** every step guarded + asserted; a failed assertion aborts nonzero with a receipt (no partial silent cutover); re-running resumes safely. Compensation is keyed on the **observed** state, not the intended one ‚Äî every partial branch of any multi-step state converges to either verified WT-only (before any retirement mutation) or verified manual-only (after any), with the roller DISABLED throughout; S2 is idempotent-to-completion.
- **Reboot durability (F3 / case-1):** post-cutover the metadata has NO watchtower and enables the roller only when `hub-roller-enabled=true` ‚áí reboot recreates Hub (resolve+launcher) + postgres (guards) + enables the roller per the flag; watchtower is NEVER relaunched ‚Äî closed AT THE SOURCE, matching greg's ¬ß6 NO-WT-RELAUNCH test (re-run **and** reboot).
