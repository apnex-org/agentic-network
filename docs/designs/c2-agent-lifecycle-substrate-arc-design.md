## Problem

Agents are unsupervised, human-launched OS processes the system cannot see into or act on. Context-window exhaustion hits mid-mission as an invisible surprise (the engineer paused/halted repeatedly during the stint) with no first-class gauge, no trigger, and no mitigation. There is NO infra primitive for the system, a peer, or an agent itself to restart / reload / reset / compact a harness: Claude Code agents run as bare `exec claude` shells (start-lily.sh / start-claude-prod.sh) with no supervisor, and the OpenCode peer (Steve) is structurally inert-until-relaunched (shim.ts:452 'no clean process-exit path... do NOT call process.exit'), so shim/config/plugin changes dead-end on a human (the LIVED 'architect-cannot-restart-greg' + 'shim/config-can't-reach-a-running-peer' gap). The Director is the restart bottleneck (A13 drag). Two fused root causes: (1) there is no control plane OUTSIDE the LLM process — the plugin/shim runs INSIDE the harness so it can never restart its own host; (2) durable working-state is fused to an ephemeral, unmeasured, exhaustible context window, and the only checkpoint primitive (save_continuation/continuationState, task-314) is queue-item-scoped (pending-action-repository-substrate.ts:258-319). The deeper risk: the org has never validated that the harnesses that matter are even MEASURABLE/CONTROLLABLE from outside — they own their own LLM loops; the shim is a guest. The org already solved this exact reconcile shape one layer down (the prod Hub runs under watchtower+systemd reconciling ACTUAL->DECLARED SchemaDef) but never turned that pattern on the agents themselves.

## Approach

Banked, staged rungs behind a hard investigation gate. W0 is the GATE (investigation, not banked value): a per-host x per-harness three-valued (GREEN/AMBER/RED) capability + fidelity verdict that, critically, pins the EXACT production execution model FIRST and proves measurability + controllability + graceful-drain ALL hold in that ONE model. L1 makes context a measured, Hub-visible, FIRST-CLASS resource with a load-bearing fidelity tag. L2 contracts a lossless, Hub-ground-truth-derived continuity handover (standalone-valuable even without containers). L3 lands the supervisor OUTSIDE the harness (containerize both harness types under the proven reconcile pattern) + the directive kind + a uniform lifecycle CLI. L4 authorizes + auto-triggers behind four hardening gates and a named staleness-proof primitive. L1/L2 are genuinely standalone banks; L3/L4 are GATED on three sibling arcs committing (C1 telemetry spine, C3-R4 renameMap governor, D-1 control plane) — recorded as a real cross-arc sequencing dependency, not assumed away. Exactly TWO new kinds (ContinuityRecord, AgentLifecycleDirective), both born under C3-R4; authority + staleness are POLICY + the existing Audit kind, not new kinds. The arc is conclusive at criteria (1)-(5); self-healing FLEET (criterion 6) is chartered-deferred, do-not-build.

MITIGATE: W0 pins the EXACT production execution model FIRST and must prove all four legs in that ONE model, not split across interactive and headless test paths.
- SPIKE-GREENS-AMBER trap (likeliest outcome): context-read + compaction reachable via statusline / PreCompact / stdin but NOT via the MCP adapter (the shim is a guest). MITIGATE: three-valued vocabulary; AMBER never silently treated as GREEN; each downstream rung carries a redesign-if-RED/AMBER precondition; the Director gate forces the fallback fork.
- AUTO-TRIGGER-ON-UNRELIABLE-GAUGE = the silent-kill / silent-work-loss hazard (the L1-measurability -> L4-safety propagation; the central A4 threat). MITIGATE: fidelity-gate (measured-only) HARD precondition + checkpoint-completeness gate + restart-rate circuit-breaker + non-gating verifier-audit — all required before L4 ships.

MITIGATE: heartbeat + work-progress two-signal primitive shipped BEFORE L4; consent-first; force-last with per-occasion Director authorization + verifier-audit.
