## Problem

The org has two coupled defects. (1) IDLE-AGENTS: there is no claimable work surface, so role-eligible agents wait on a manual dispatcher instead of self-picking ready work — a tele-13/tele-6 leak. (2) SCATTERED CONTROL: the implicit Hub-apiserver (resources=the locked kinds, verbs=the policy tools) is fronted by a drift-prone scatter of ad-hoc tools (get-*.sh, calibrations.py, hub-snapshot.sh, psql-cookbook) with no single sovereign, RBAC-scoped, audited surface — a tele-3/tele-10 leak. D-1 reframes the fix as a first-class root-level cli/+api/ control plane (a kubectl/apiserver analog) exposing observe + CLAIM + LEASE + ACTUATE + evidence-to-close. The deep hazard, surfaced by Steve's verifier critique, is OVER-CLAIM: a read surface that calls itself a control plane is just a nicer status page. A control plane requires ACTUATION + leases/claims + backpressure + CROSS-ADAPTER PARITY — and the parity leg is genuinely hard because steve/OpenCode is a guest inside the harness LLM loop with no proven proactive claim/emission seam.

## Approach

A GOVERNED-CONTRACT + COORDINATION-PROTOCOL + CONFORMANCE-REGIME arc, staged to EARN the control-plane label rung by rung and forbidden from asserting it early. The over-claim guard is operationalized as an explicit 4-property ACCEPTANCE CHARTER (ACTUATION + LEASES/CLAIMS + BACKPRESSURE + CROSS-ADAPTER PARITY) authored in R1; the arc may NOT be called a control plane until R3 discharges all four. A front-loaded W0 feasibility SPIKE (shared with C2-W0) resolves the deepest risk first: it distinguishes VISIBILITY-parity (steve SHOWN) from ACTUATION-parity (steve CLAIMS) and thereby sets the HONEST scope the label is earned at — a heartbeat/idle proxy may never silently pass the parity dogfood. R1 ships a governed READ plane (honestly status-page-grade) with the api-resources/api-verbs contract DERIVED at runtime from the live SchemaDef inventory + the verb registry (auto-correcting the 20/21/22 kind-count drift; the only tele-10-honoring shape), and folds the tool-scatter into oisctl by EXTENDING the existing mod.core (mission-75) bash CLI lib. R2 is the KEYSTONE: a reference-only WorkItem kind + claim/lease/ack verbs — the literal idle-agents fix (self-PICK, no dispatcher), with Hub-ENFORCED backpressure and wedged-agent thrash detection so it recovers the AGENT, not just the work. R3 discharges parity at the scope the W0 spike licenses. R4 converges org-health onto ONE durable telemetry spine. RACI per the arc constraint: the architect OWNS the resource/verb CONTRACT (R1 conventions + R2 protocol spec); greg LEADS construction-Design from R2. Mission class = structural-inflection → walkthrough retrospective + substrate-self-dogfood gate (coordinate the NEXT arc's missions THROUGH the queue). Self-PICK (pull) only; proactive-push, C2 lifecycle-actuation, verifier-gating, and the governance-fold are all chartered-deferred (named for honesty, built elsewhere).
