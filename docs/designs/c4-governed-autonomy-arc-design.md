# C4 — Governed Autonomy Substrate · Arc Design (DRAFT)

**Status:** DRAFT — Ultracode `arc-design-panel` output (run wf_1cf1556b-719; 3-angle panel + adversarial verify + synth). Pending Director gate. Date 2026-06-21.

**Arc:** C4 — Governed Autonomy Substrate (codify-what-worked; record-first; NO enforcement until recurrence)

## Problem
Stint-1 proved an autonomous operating model end-to-end (DR-001..012 Director-ratified; hard-lines HELD — DR-008 refused hand-recreating the Hub container, agreed prod-infra-ops as case-by-case-not-blanket; DR-002 prod-deploy-authority blessed and both autonomous deploys were Steve-verified; the cross-lineage verifier backstop fired when convened). What is broken is NOT safety — it is REPEATABILITY + REVIEWABILITY. The model lived as a verbal authority handshake plus an uncommitted DR log that does not exist anywhere in the repo (confirmed by grep: zero autonomy-charter / DR-NNN / AuthorityGrant / DecisionRecord artifacts). The verifier backstop fired only because the architect CHOSE to convene Steve (recall-blind by construction — the exact precision-trustworthy-but-recall-blind finding that birthed verifier-role.md, calibration #87). And the Director's review was markdown archaeology against working memory. So the next stint cannot be reliably re-run, and its decisions cannot be queried — they can only be re-narrated, which re-imports the LLM-state-fidelity-drift the org built ledgers to defeat. The honest fix is to write down exactly what worked, make it queryable, make the next stint reviewable, and STOP — not to pre-build an enforcement substrate (deny-set membrane, auto-revert, verifier-gating, harness pre-authorization) for a prose-discipline failure class observed ZERO times.

## Approach
This is a CODIFICATION + DURABILITY arc, not a control arc — the org's own "structurally eliminate RECURRING defect classes" discipline applied honestly: build the wall after the breach, not before. Two banked rungs that each pay out alone; the second is a backing-store swap of the FIRST's shapes, not a re-model. R1 (S, zero Hub code, one session) clones the proven calibration-ledger DX (docs/calibrations.yaml + scripts/calibrations/calibrations.py + skills/survey mold + ADR-030) into an autonomy charter + DR-ledger with an ADVISORY pre-flight authority-check; the decision-protocol [triangulate-vs-teles -> record-DR -> execute-or-defer] IS the DecisionRecord schema, so discipline becomes queryable rather than recalled (the tele-10 drift-defeat). R2 (L — full per-kind substrate tax, NOT pattern-replication) promotes the SAME two shapes to durable K8s-envelope Hub entities plus a Director batch-ratify query — RECORD + OBSERVE only. Posture honoring every constraint literally: prose-first, record-first, ADVISORY-only, NO enforcement substrate. The verifier stays STRICTLY advisory by construction (its verdict attaches as a non-gating create_audit_entry; the DecisionRecord finalizes immediately; no pending-integrity-check state, no verifier reopen-authority, no create_review) — so verifier-role.md §1/§2.3 is never reopened and no re-ratification dependency is incurred. The Ultracode/Auto-Mode governance folds in as written PROSE doctrine (no blanket pre-auth; Auto Mode = one-time-per-workflow consent; any Hub-grant TIGHTENS-never-loosens the local harness ceiling). Verifier-checks-as-queued-work-items is a forward-pointer to the C1 work-control-plane arc, NOT built here. Three corrections from the adversarial pass are load-bearing in this synthesis: (1) the DR-ledger write-discipline INVERTS the calibration rule — DRs are author-autonomous-in-flight, only the ratified status is Director-gated (so we must NOT copy the calibration "never LLM-autonomous" line into CLAUDE.md); (2) the seed is a PROVENANCE exercise — reconstruct DR-001..012 from primary artifacts that DO exist (Hub audit entries, thread transcripts, deploy run logs, PR history) with a source-ref per DR, and DROP the unverifiable "131-vs-400 lines" counter-claim as equally unevidenced; (3) the charter must be LOADED to be load-bearing — wire it into CLAUDE.md cold-pickup load-order + an architect-runtime.md index entry, not just a pointer, else it repeats the engineer-runtime-rules-invisible shelfware class.

      title / context: str
